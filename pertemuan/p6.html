PERTEMUAN 6

Thread dan Multithreading

Pengertian Thread

Thread merupakan sebuah status eksekusi dalam sistem operasi seperti running, ready, dan lainnya, sehingga sering disebut sebagai lightweight process. Thread merupakan unit dasar dari penggunaan CPU yang terdiri dari thread_id, program counter, register set, dan stack. Dalam satu proses, beberapa thread dapat berjalan secara bersamaan dan berbagi code section, data section, serta sumber daya sistem operasi yang sama.

Dibandingkan dengan proses, thread memiliki overhead yang lebih kecil karena tidak memerlukan alokasi sumber daya yang terpisah secara penuh. Hal ini membuat pembuatan dan pengelolaan thread menjadi lebih cepat dibandingkan dengan proses. Oleh karena itu, thread sering digunakan pada aplikasi yang membutuhkan kinerja tinggi dan respons cepat.

Konsep Multithreading

Thread memungkinkan komputer untuk menjalankan dua atau lebih tugas dalam waktu yang bersamaan. Sedangkan multithreading merupakan teknik pembagian pekerjaan ke dalam bagian-bagian kecil yang dieksekusi secara cepat sehingga terlihat berjalan secara paralel. Dengan pendekatan ini, sistem dapat memanfaatkan sumber daya CPU secara lebih optimal.

Pada sistem modern, multithreading sangat penting untuk mendukung aplikasi yang kompleks dan interaktif. Aplikasi seperti web browser, sistem basis data, dan aplikasi multimedia sangat bergantung pada multithreading agar dapat menjalankan berbagai fungsi secara simultan tanpa mengganggu pengalaman pengguna.

Single-threading dan Multi-threading

Berdasarkan jumlah thread yang digunakan, terdapat dua jenis pendekatan eksekusi proses, yaitu single-threading dan multi-threading. Pada single-threading, sebuah proses tradisional atau heavyweight process hanya memiliki satu thread yang berfungsi sebagai pengendali utama.

Pada pendekatan ini, jika thread tersebut mengalami hambatan seperti operasi input/output, maka seluruh proses akan terhenti sementara. Hal ini menyebabkan efisiensi sistem menjadi rendah, terutama pada aplikasi yang membutuhkan banyak interaksi dengan pengguna.

Sebaliknya, pada multi-threading, sebuah proses memiliki banyak thread yang dapat dijalankan secara bersamaan. Jika satu thread mengalami penundaan, thread lain tetap dapat berjalan sehingga sistem tetap responsif dan efisien.

Karakteristik Multithreading

Multithreading adalah teknik di mana suatu proses dibagi menjadi unit eksekusi yang lebih kecil yang disebut thread dan dijalankan secara bersamaan. Thread juga disebut sebagai proses ringan karena membutuhkan sumber daya yang lebih sedikit dibandingkan proses penuh.

Konkurensi atau paralelisme dicapai dengan membagi proses menjadi beberapa thread yang dapat dijadwalkan oleh sistem operasi. Sistem operasi akan mengatur eksekusi thread tersebut sehingga setiap thread mendapatkan kesempatan menggunakan CPU secara adil.

Manfaat Penggunaan Multithreading

Penggunaan multithreading dapat meningkatkan kinerja dan responsivitas sistem. Beberapa thread dapat berbagi sumber daya CPU, memori, dan perangkat input/output dari satu proses yang sama.

Contohnya pada peramban web, setiap tab dapat dijalankan sebagai sebuah thread sehingga pengguna dapat membuka banyak halaman tanpa membuat aplikasi menjadi lambat. Pada aplikasi pengolah kata seperti Microsoft Word, satu thread memformat teks sementara thread lain memproses input pengguna.

Model Multithreading

Dalam sistem operasi, terdapat beberapa model multithreading yang digunakan untuk mengatur hubungan antara thread pengguna dan thread kernel. Model-model tersebut meliputi model banyak ke banyak, model banyak ke satu, dan model satu lawan satu.

Pemilihan model multithreading sangat memengaruhi kinerja dan fleksibilitas sistem operasi. Setiap model dirancang untuk memenuhi kebutuhan tertentu dan memiliki kelebihan serta keterbatasan masing- masing.

Kesimpulan

Dengan memahami konsep thread dan multithreading, mahasiswa diharapkan mampu memahami bagaimana sistem operasi menjalankan banyak tugas secara bersamaan, meningkatkan efisiensi pemrosesan, serta menjaga kinerja dan stabilitas sistem secara keseluruhan.
